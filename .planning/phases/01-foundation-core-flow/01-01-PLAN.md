<task type="auto">
  <name>Task 1: Initialize Vite + React + TypeScript project with all dependencies</name>
  <files>
    package.json
    vite.config.ts
    tsconfig.json
    tsconfig.node.json
    src/vite-env.d.ts

  </files>
  <action>
Create a new Vite project with React + TypeScript template:

1. Run `pnpm create vite@latest . -- --template react-ts` to initialize in current directory (or scaffold manually if directory has files)

2. Install all dependencies from research:
```bash
pnpm add react-router react-hook-form zod @hookform/resolvers
pnpm add -D vite-plugin-pwa vite-plugin-mkcert tailwindcss @tailwindcss/vite @types/node
```

3. Configure vite.config.ts with:
- @vitejs/plugin-react
- @tailwindcss/vite
- vite-plugin-mkcert for HTTPS localhost
- vite-plugin-pwa with manifest:
  - name: "DashDecoder"
  - short_name: "DashDecoder"
  - theme_color: "#2563EB" (blue accent from landing page)
  - background_color: "#ffffff"
  - display: "standalone"
  - icons: 192x192 and 512x512 PNGs
  - devOptions.enabled: true
  - workbox.runtimeCaching: NetworkOnly for api.openai.com

4. Configure tsconfig.json with:
- strict: true
- noUnusedLocals: true
- noUnusedParameters: true
- noUncheckedIndexedAccess: true
- baseUrl: "."
- paths: { "@/*": ["./src/*"] }

5. Configure resolve.alias in vite.config.ts:
- "@": path.resolve(__dirname, "./src")

6. Update src/vite-env.d.ts to include:
```typescript
interface ImportMetaEnv {
  readonly VITE_OPENAI_API_KEY: string
}
```

7. Create .env.example with:
```
VITE_OPENAI_API_KEY=your_openai_api_key_here
```

Do NOT install shadcn/ui yet - that requires the base config to exist first.
  </action>
  <verify>
Run `pnpm dev` - should start HTTPS server on https://localhost:5173
Run `pnpm build` - should complete without errors
  </verify>
  <done>
Vite dev server runs with HTTPS, TypeScript strict mode enabled, PWA manifest generated
  </done>
</task>

- CSS variables: Yes (required for dark mode)

2. Verify components.json has:
- aliases.components: "@/components"
- aliases.utils: "@/lib/utils"

3. Update src/index.css for dark mode:
- Add `@import "tailwindcss";` at top
- Ensure :root and .dark CSS variables are present (shadcn adds these)
- Add custom blue accent variable if needed: --accent: 217.2 91.2% 59.8%;

4. Create clean src/App.tsx:
```tsx
export default function App() {
  return (
    <div className="min-h-screen bg-background text-foreground">
      <main className="container mx-auto px-4 py-8">
        <h1 className="text-3xl font-bold">DashDecoder</h1>
        <p className="text-muted-foreground mt-2">AI-powered dashboard warning light diagnosis</p>
      </main>
    </div>
  );
}
```

5. Update src/main.tsx:
```tsx
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'

import App from './App'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <App />
  </StrictMode>,
)
```

6. Test dark mode by adding `class="dark"` to html element temporarily - colors should change.

7. Add a shadcn button to test the setup:
